# Workflow: Plan Feature

## Use When

- a feature spans product, mobile, backend, and possibly AI

## Steps

1. Read `AGENTS.md`
2. Read `.agent/rules/10-medicineapp-core.md`
3. Use `medical-product-architect`
4. Map the feature across:
   - user journey
   - Flutter screens
   - Node API changes
   - Python or pipeline changes if any
   - test impact
5. Produce a phased plan with clear acceptance criteria

## Deliverable

- a concise implementation plan tied to actual repo paths
